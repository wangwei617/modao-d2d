# 墨刀 AI 产品优化需求文档（营收与用户增长）

> **文档类型**：需求总纲（基于数据与用户画像的合成结论，供立项与排期引用）  
> **版本**：v1.0  
> **依据文档**：[墨刀AI近30天使用情况分析_2026-04-16.md](./墨刀AI近30天使用情况分析_2026-04-16.md)、[墨刀AI用户画像.md](./墨刀AI用户画像.md)、[数据结构说明.md](./数据结构说明.md)  
> **数据口径**：近 30 日任务约 **124,638** 条、去重用户约 **87,048**；行为字段以 `mode`、`status`、`name`+`plan` 代理意图为主。

---

## 1. 背景与目标

### 1.1 背景

- 用户将墨刀 AI 主要用作 **「一句话 → 可演示交付物」**（HTML 原型 + MD 说明为主），**大屏 / C 端 / 后台 / 界面复刻** 四类合计约 **六成** 需求。  
- **约 75%** 会话为 **`mode=general`**（未挂二级标签），首页「卡片结构」对主群体触达有限。  
- 主路径 **`finished` 约 93.7%**，体验基线较好；**增长与营收瓶颈**更集中在：**价值闭环（进主站/协作/导出）、意图与入口匹配、澄清与长尾承接、计费触点合理性**。

### 1.2 产品目标（可量化方向）

| 目标维度 | 方向性指标（需在立项时细化基线与目标值） |
|----------|-------------------------------------------|
| **用户增长** | 新用户 **首任务完成率**、**7 日回访**、**人均任务数**提升；`general` 下 **有效场景命中率** 提升。 |
| **营收增长** | **转墨刀项目/编辑器** 转化率、**深度/增值能力** 使用率与付费渗透率、**团队/企业** 相关功能使用率、**错配与无效调用** 导致的客诉/退款下降。 |
| **效率与健康** | **`clarifying`→`finished` 转化**、「其他」类意图 **`finished` 向大盘靠拢**、关键 `mode` **队内错配率**下降。 |

### 1.3 设计原则

1. **先深化主场景，再控制扩容节奏**：避免卡片膨胀、错配与「其他」低完成率被放大。  
2. **以 `general` 为主设计第一公民**：占位、案例、意图识别、发送后推荐标签与扩容同等重要。  
3. **任何新场景必须带「商业闭环」指标上线**：完成率、二次使用、转项目/导出，至少绑定其一。

## 2. 关键结论摘要（驱动需求的证据）

| 结论 | 数据/行为要点 | 对需求的含义 |
|------|----------------|----------------|
| 主需求集中 | Top4 场景合计约 **62%** | 资源优先投入 **大屏、C 端、后台、复刻** 的端到端体验与变现。 |
| 通用入口主导 | `general` **75.19%** | 首页与对话 **强引导、强案例、强意图分流**，不能只优化「点卡片」路径。 |
| 错配伤付费心智 | `image_generation` 队内错配 **约 55.3%**；`product_planning` **约 42.6%** | **P0**：入口、文案、发送前确认、标签继承策略。 |
| 「其他」桶脆弱 | 约 **8.6%** 任务且 `finished` **约 78%** | **P0/P1**：意图识别、异常任务、标题失败与重试。 |
| 澄清是漏斗 | `clarifying` **约 3.79%** | **P1**：默认方案一键确认、进度感知、减少空转。 |
| 极速为主 | `express` **约 89.3%**；Token **P90** 远高于 P50 | **P1**：深度/重任务与 **计费/额度** 展示一致，避免「快」预期与账单冲突。 |

## 3. 需求清单总览（优先级）

| 编号 | 需求主题 | 优先级 | 主目标（增长 / 营收） |
|------|----------|--------|-------------------------|

| R2 | 「其他」类意图与异常任务体验 | **P0** | 增长（完成率、留存） |

## 4. 需求详述

### R1 二级能力与真实意图一致（错配治理）【P0】

**问题**  
部分用户挂载二级 `mode` 后，实际在 `name`/`plan` 中表达的是另一类任务（如挂生图却描述页面/PRD；挂产品规划却直接要界面）。全站任务占比不高，但**在少数 `mode` 子集内错配率极高**，易导致额度浪费与放弃付费。

**用户价值**  
少点错、少误会、一次对话就能对上「我要的那种输出」。

**商业价值**  
降低无效 Token 与客诉；提高 **付费额度「获得感」**；为按场景打包售卖扫清体验障碍。

**范围建议**  

1. **发送前意图检测**：当文本强偏「页面/HTML/PRD」与当前 `image_generation` 冲突时，**弹窗或横条**建议切换至原型/文档。  
2. **`product_planning` 卡片**：副标题与发送前说明明确 **默认产出形态**（路线图/里程碑 vs 高保真页），提供 **一键切到原型**。  
3. **新会话默认策略**：评估 **是否继承上一会话标签**（可配置或默认不继承），降低隐性错配。  
4. **文档类 `documentation`**：检测到「强页面实现」信号时，引导切至原型/交互方案。

**验收指标**  

- `image_generation`、`product_planning` **队内错配率**环比下降（需埋点定义「错配纠正」与「用户确认切换」）。  
- 上述 `mode` 下 **`finished` 与满意度代理指标**（若有 NPS/差评标签）不下降。

**依赖**  

- 建议落库 **首条用户明文**、**发送前最终 `mode`**、**是否经系统纠偏**，见 R7。

---

### R2「其他」类意图与异常任务体验【P0】

**问题**  
规则未命中的「泛需求」桶占比约 **8.6%**，且 **`finished` 约 78%**，显著低于大盘；桶内存在导入、评审、单点优化及**异常标题**等混合情况。

**用户价值**  
说不清时也能被接住；异常时能重试、能继续，而不是黑箱失败。

**商业价值**  
减少「一次失败即流失」；为后续 **按次付费、会员权益** 保留信任。

**范围建议**  

1. **标题生成失败 / 异常任务**：明确错误态、**重试**与**降级为纯对话修改**路径。  
2. **碎片化意图**：对「导入 / Figma / 评审改进」等提供 **最短官方路径**（链路到现有工作台能力）。  
3. **意图过弱时**：主动给出 **3～5 个可选场景标签** 让用户点选，减少空泛输入。

**验收指标**  

- 「其他」桶或等价标签下 **`finished` 提升**、**同会话重试成功率**。  
- 异常类标题占比或 **error 关联会话** 可监控下降。

---

### R3 Top4 场景深化（可交付、可进主站）【P0】

**问题**  
用户已为 **大屏、C 端、后台、复刻** 付出大量注意力；若停留于「能看的 HTML」，**与墨刀编辑器、协作、评审、版本的主商业闭环** 断裂，营收天花板低。

**用户价值**  
同一需求下：**少澄清、多页面一致、能进团队流程、能对外交付**。

**商业价值**  

- **转项目 / 转原型文件**：带动 **席位、空间、协作** 等核心营收。  
- **深度生成 / 行业包**：带动 **增值档位、企业套餐**。

**范围建议**  

1. **一键导入墨刀**：多文件 HTML → 墨刀项目/页面结构（含组件映射策略说明）。  
2. **Top4 行业/场景模板**：如能源大屏、医院运营、CRM 后台、复刻工作流（上传 → 对齐 → 迭代）。  
3. **交付物结构标准化**：README、页面清单、设计说明与 **导出 zip** 等，便于乙方/团队交付。  
4. **深度模式与 express 的边界产品化**：与 **额度、预计时间、文件数** 一致，避免预期撕裂（对齐当前 Token 长尾数据）。

**验收指标**  

- **AI 任务 → 打开编辑器 / 创建项目** 的转化率与次数。  
- Top4 相关场景的 **二次任务率、7 日回访**。  
- 深度/增值能力 **使用率与 ARPU 代理指标**（具体计费模型由商业团队定义）。

---

### R4 `general` 路径：占位、案例、意图与标签推荐【P1】

**问题**  
**约 75%** 用户不挂二级标签；若占位与案例与真实高频场景（后台、大屏、复刻、PRD、小程序）脱节，**增长依赖自然语言「撞大运」**。

**用户价值**  
打开即知道「可以怎么说」；发完后系统帮我对上能力。

**商业价值**  
提高 **首任务成功与回访**；为 **场景化运营位**（推荐卡片、热门模板）提供数据选品依据。

**范围建议**  

1. 输入框 **占位与生成案例** 与 Top 场景及 `name` 高频主题对齐。  
2. **发送后轻量推荐**：「更像 PRD / 更像大屏」时推荐挂载或切换路径（不阻断发送）。  
3. **界面复刻**与上传在 `general` 下同样 **短路径可达**。

**验收指标**  

- 新用户 **首条任务 `finished` 率**、**首周任务数**。  
- `general` 下 **显式挂载率**或 **场景归类命中率**（若可统计）。

---

### R5 澄清阶段（`clarifying`）体验与转化【P1】

**问题**  
**约 3.79%** 任务处于澄清态；若轮次多、价值感弱，用户会认为「慢、卡住」，与 **约 89% express** 心智冲突。

**用户价值**  
对齐需求时 **有进度、有默认、可一键继续**。

**商业价值**  
缩短 **同等价值下的 Token 与时间**；提高 **`finished` 转化**，间接保护额度商业模型。

**范围建议**  

1. **默认方案一键确认**（与现有快捷 chip 能力对齐并量化点击）。  
2. **阶段化进度**（对齐中 / 生成前 / 生成中）。  
3. 对高频场景预置 **澄清问题模板**，减少开放式追问。

**验收指标**  

- `clarifying` → `finished` **转化率**、**平均停留时长**、**中途离开率**。

---

### R6 场景扩容（受控：行业包、模板库、长尾）【P2】

**问题**  
长尾（图表、技术方案、用例、竞品等）**单量不大但专业**，仅靠首页堆卡片 **ROI 低且易稀释主心智**。

**用户价值**  
「我专业需求也能找到官方路径」。

**商业价值**  
扩大 **可计费场景表面积**；行业包、模板包可 **单独售卖或与会员捆绑**。

**范围建议**  

1. **模板库 / 搜索 / 最近使用** 承接长尾，首页保留 **有限强运营位**。  
2. **新卡片上线门槛**：必须绑定 **完成率、转项目率、使用量** 之一达标，否则合并或下线。  
3. **扩容与 R1 错配治理同步**：新卡片文案与发送前逻辑沿用同一套意图规则。

**验收指标**  

- 新场景 **使用量、完成率、复访**；对大盘 **`finished` 无显著负向**。

---

### R7 数据与埋点补全（平台）【P0】

**问题**  
当前分析缺乏：**一级 Tab**、**首条用户明文**、**转编辑器/转项目**、**付费档位** 等，难以把优化与 **营收归因** 绑死。

**范围建议**  

1. 埋点：**`l1_tab`**（或等价）、**最终生效 `mode`**、**首条 `user_message` id**、**clarifying 进出时间**、**一键转项目**、**导出**。  
2. 数仓：与 **订单/席位/额度** 可关联的用户与空间维度（脱敏与合规前提下）。  
3. Metabase 看板：**错配代理指标、Top4 漏斗、转项目漏斗**。

**验收指标**  

- 核心漏斗 **可查、可按版本对比**；需求 R1～R6 均能 **挂指标上线**。

---

## 5. 非目标（本版明确不做或暂缓）

- 不追求「首页二级卡片数量」作为成功标准。  
- 不以 **全站对话式万能助手** 替代 **可演示交付 + 进主站** 的主叙事（除非有单独品牌战略）。  
- 不在 **未治理错配** 前大规模上新计费点（易引发客诉）。

## 6. 实施节奏建议（与研发排期对话用）

| 阶段 | 内容 | 说明 |
|------|------|------|
| **Wave 0** | R7 埋点最小集 + 错配与转项目漏斗可看 | 与 Wave 1 并行启动 |

| **Wave 3** | R6 受控扩容 | 依赖 Wave1 指标稳定 |
