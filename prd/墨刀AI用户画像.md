# 墨刀 AI 用户画像（基于近 30 天 Metabase 工作台数据）

> **文档目的**：在不做线下调研的前提下，从**任务级行为数据**反推「谁在用什么方式、为了什么、做到哪一步」，形成可指导产品与运营的**合成用户画像**。  
> **主要依据**：[墨刀AI近30天使用情况分析_2026-04-16.md](./墨刀AI近30天使用情况分析_2026-04-16.md)（下称《近30天分析》）；字段与 UI 对齐见 [数据结构说明.md](./数据结构说明.md)。  
> **时间窗口**：`created_at` 在 **2026-03-17～2026-04-15**（30 个自然日，含）。  
> **数据实体**：一行 ≈ 一次 **AI 工作台会话/任务**（非单次模型调用），见《数据结构说明》第 1 节。

---

## 1. 画像的数据边界（读前必读）

下列信息**当前 CSV 无法直接给出**，本文**不臆测**年龄、城市、职级、公司规模等人口统计字段：

- 用户年龄、性别、地域、行业标签  
- 是否付费、套餐档位、团队角色（产品/设计/研发）  
- **一级 Tab 点击路径**（推荐 / 原型设计 / 产品策划等**未落库**）  
- **首条用户原文**（分析中用 `name` + `plan` 前缀替代）

下列信息**可直接或间接从数据推导**：

- **行为**：是否挂载二级 `mode`、`execution_mode`、任务 `status`、产出文件类型  
- **意图代理**：系统标题 `name`、计划 `plan` 关键词、场景归类规则  
- **强度**：`token_usage` 分布（P50 / P90）、澄清态占比

因此本文是 **「行为 + 意图代理」画像**，适合与后续问卷、访谈、埋点（`l1_tab`、`user_query`）叠加以逼近真实人物。

## 2. 群体轮廓：有多大、多活跃


| 指标                | 数值          | 解读                                                     |
| ----------------- | ----------- | ------------------------------------------------------ |
| 30 日内任务行数         | **124,638** | 高活跃的需求侧信号                                              |
| 30 日内去重 `user_id` | **87,048**  | 广覆盖的独立使用者规模                                            |
| 粗算人均任务数（任务/人）     | **约 1.43**  | 均值被「只来一次」用户拉低；存在大量**多任务重度用户**与大量**低频尝鲜**的混合分布（需留存曲线验证） |


**一句话**：墨刀 AI 在近一个月内触达 **约 8.7 万独立用户**，以**离散任务**为主（原型/文档/大屏等「一单一交付」心智强）。

## 3. 行为心智：怎么进来、怎么跑任务

### 3.1 「先说话」还是「先点能力」


| 路径               | 占比         | 数据表现           | 用户心智推断                                                             |
| ---------------- | ---------- | -------------- | ------------------------------------------------------------------ |
| **通用入口**（未挂二级标签） | **75.19%** | `mode=general` | **「我直接描述问题，让 AI 自己理解」**；对首页结构依赖低，把墨刀 AI 当**通用产出工作台**。              |
| **显式挂载二级能力**     | **24.81%** | `mode≠general` | **「我要用某一种官方能力」**；其中 **原型类 `prototyping` 占绝对大头（22.77%）**，其余垂类各 <2%。 |


**画像含义**：主流用户是 **「自然语言优先」型**；愿意先选卡片、再输入的，多数是 **强原型心智** 或**明确垂类需求**（PRD、用例、竞品等）。

### 3.2 速度与深度

- `**execution_mode=express` 约 89.3%**：默认偏好 **快、可迭代**，与首页「极速」一致。  
- `**deep` 约 10.7%**：少数 **重任务 / 高要求** 或愿意多花时间换更完整链路。  
- **Token：P50 ≈ 1.1 万，P90 ≈ 21 万**：存在明显 **长尾重会话**（多轮、多文件、长 plan），对应「项目型」使用者。

### 3.3 任务生命周期与情绪代理


| `status`     | 占比         | 对画像的暗示                                          |
| ------------ | ---------- | ----------------------------------------------- |
| `finished`   | **93.74%** | **结果导向、可完成感强**；产品主路径整体可信。                       |
| `clarifying` | **3.79%**  | 相当一部分用户经历 **「先对齐再生成」**；若体验拖沓，会伤害「快」心智。          |
| `paused`     | **2.20%**  | 主因 **用户主动暂停**（见分析文档）；多为**打断、改主意、或暂时满意**，不一定是失败。 |
| `error`      | **0.27%**  | 技术失败感知面小，但个体遇到时伤害大。                             |

## 4. 动机与场景：用户「来干什么」（JTBD 视角）

结合《近30天分析》**Top10 场景**（合计约 **91.1%** 任务）与 `**name` 高频标题**，可将核心动机归纳为五类 **Jobs to be Done**：


| JTBD 类型         | 典型场景占比（提示量级）                   | 用户想达成的结果                                   |
| --------------- | ------------------------------ | ------------------------------------------ |
| **A. 可演示的界面交付** | Top4 合计 **约 62%**（大屏+C端+后台+复刻） | 尽快得到 **能打开、能翻页、能给人看** 的 HTML 原型（常配 MD 说明）。 |
| **B. 需求与协作材料**  | PRD 等 **约 6～7%** + 长尾文档        | 产出 **可评审、可挂链接** 的 PRD/规格/流程说明。             |
| **C. 营销与视觉物料**  | 生图等 **约 5～6%**                 | **海报、KV、Logo、配图**，偏市场/运营侧。                 |
| **D. 小工具与单页**   | 单页小组件等 **约 4%**                | **登录页、计算器、首页** 等「小而快」的验证或作业。               |
| **E. 汇报与对外表达**  | PPT 等 **约 2%**                 | **路演、周报、培训** 的结构化演示材料。                     |


**高频标题透露的行业/题材**（来自《近30天分析》附录）：企业后台、CRM、数据大屏、景区/文旅 App、教育、医疗、生鲜电商、**界面复刻**、测试用例、PRD 等——用户群 **B 端数字化 + 互联网产品 + 教育医疗垂直** 交叉明显。

## 5. 合成人物画像（便于评审与故事板）

以下人物为 **基于数据聚类的合成画像**（非真实个体），用于需求讨论与首页改版对齐。

---

### 画像 A：**「一句话交付型」通用探索者（主群体）**


| 维度       | 描述                                                                          |
| -------- | --------------------------------------------------------------------------- |
| **代表行为** | `**mode=general` 为主**；直接输入需求，不先点二级卡片。                                       |
| **动机**   | 今天就要 **能演示的东西**：大屏、后台、小程序、复刻某界面。                                            |
| **成功标准** | `finished`；能下载/预览 **多文件 HTML**；必要时有一份 **MD** 讲清楚。                           |
| **痛点**   | 意图分散时落入 **「其他」桶（约 8.6% 任务）**，该桶 `**finished` 仅约 78%**；异常标题、导入/评审类碎片化需求体验不稳。 |
| **产品启示** | **占位示例、生成案例、发送后轻量推荐挂载标签**，降低「说不清」比例。                                        |


---

### 画像 B：**「显式原型型」产品经理 / 设计师**


| 维度       | 描述                                                                         |
| -------- | -------------------------------------------------------------------------- |
| **代表行为** | `**prototyping` 约 22.8%**；或挂在原型下但文本更泛（「显式 prototyping 未细分桶」约 **2.57%**）。   |
| **动机**   | 在 **明确原型能力** 下推进 **多页面、多角色、复杂流程**。                                         |
| **成功标准** | 多文件、可迭代；`finished` 在显式原型细分场景仍高，但 **「仅挂 prototyping 标签」桶完成率约 84%**，更重、更易多轮。 |
| **痛点**   | 有时实际想要 **PRD / 导图 / PPT / 生图**，仍挂在原型下（错配见画像 F）。                            |
| **产品启示** | 对话内 **能力切换建议**；重任务时引导 **深度模式**。                                            |


---

### 画像 C：**「运营与数据叙事者」**


| 维度       | 描述                                                        |
| -------- | --------------------------------------------------------- |
| **代表行为** | 场景 **「数据可视化 / 运营大屏」约 17%**；标题常带节能减排、智慧医院、能源等 **行业 + 大屏**。 |
| **动机**   | 用一块屏 **讲清 KPI、趋势、告警、业务分区**，对内汇报或对外展示。                     |
| **成功标准** | `**finished` 约 97.7%**；中位约 **3 个文件**；HTML + MD 即可闭环。      |
| **痛点**   | 与画像 A 重叠：若描述含糊，易掉入「其他」或澄清轮次过多。                            |
| **产品启示** | **大屏模板、行业案例、指标组件词典**；澄清阶段给 **默认布局一键确认**。                  |


---

### 画像 D：**「文档与流程写作者」**


| 维度       | 描述                                                             |
| -------- | -------------------------------------------------------------- |
| **代表行为** | PRD/文档场景 **约 6.7%**；`documentation` **约 1%**；长尾含用户调研、竞品、技术方案等。 |
| **动机**   | 要 **可评审、可版本化** 的文字与结构，而非仅一屏 UI。                                |
| **成功标准** | **MD 为主**；`finished` 约 **94%** 量级。                             |
| **痛点**   | **错配**：挂 PRD 标签却在要 **高保真页面**（队内约 **2.3%**）；或反向——想做文档却走通用/原型。   |
| **产品启示** | 发送前 **意图分流**；文档类输出 **目录模板与导出路径** 显性化。                          |


---

### 画像 E：**「视觉物料需求者」**


| 维度       | 描述                                                                                            |
| -------- | --------------------------------------------------------------------------------------------- |
| **代表行为** | 生图场景 **约 5.5%**；`image_generation` **约 0.08%** 任务量（绝对量小）。                                     |
| **动机**   | 海报、Logo、主视觉、活动物料；`**finished` 约 98.4%**，满意度潜力高。                                               |
| **痛点**   | **严重错配**：`image_generation` 队内 **约 55.3%** 会话文本更像在做 **原型/文档/PPT**——典型 **「点错入口」或「默认记住了上次标签」**。 |
| **产品启示** | **卡片文案与发送前确认** 优先于继续堆功能；考虑 **新会话默认不继承标签**。                                                    |


---

### 画像 F：**「入口混淆型」（需产品特别关照的少数派）**


| 维度       | 描述                                                                             |
| -------- | ------------------------------------------------------------------------------ |
| **代表行为** | 挂载二级 `mode` 后，文本意图与标签 **规则不一致**（全站约 **0.17%** 任务，占挂载会话 **0.69%**）。             |
| **高发错配** | 除生图外，`**product_planning` 队内错配约 42.6%**（要界面却点规划）；`**user_research` 约 6.6%** 等。 |
| **动机**   | 不是「不会用」，而是 **卡片心智与当下任务不一致** 或 **路径惯性**。                                        |
| **产品启示** | 《近30天分析》第 6.5 节：**发送前确认、副标题澄清、继承标签可配置**。                                       |

## 6. 用户与产品的关系：信任、摩擦与增长暗示


| 维度        | 观察                         | 对画像的补充                                       |
| --------- | -------------------------- | -------------------------------------------- |
| **信任**    | 高 `finished`、高 express 采用率 | 用户愿意把「一阶段工作」交给 AI **快速出稿**。                  |
| **摩擦**    | 「其他」桶低完成率、错配、澄清占比          | **表达成本**与**入口认知**仍是主要摩擦，而非模型「完全不能做」。         |
| **增长/渗透** | 8.7 万人在 30 日内至少一次任务        | 适合按 **场景召回**（大屏/后台/复刻）做运营与模板增长，而非仅泛称「AI 助手」。 |

## 7. 画像使用建议（对内）

1. **需求评审**：用 **画像 A + C** 压首页默认路径；用 **画像 D + E** 压垂类卡片与错配治理。
2. **指标看板**：除 `finished` 外，为 **画像 A** 单开 `**clarifying` 转化率**；为 **画像 E/F** 单开 **错配纠正率**（推荐切换标签点击率）。
3. **下一轮画像升级**：补齐 `**org_cid` / 空间类型**、**付费分层**、**首条 user_message`**、**一级 Tab 埋点** 后，将本文「合成人物」替换为 **分群统计画像 + 少量访谈引用**。
